# 핵심 발견 / 해석

수치는 `01-results.md`, 측정 신뢰 근거는 `02-methodology.md`.

## 1. MTP는 전 dense 모델·정밀도에서 가속 + 정확도 무손실

- 31B fp8 **2.64x**, 12B BF16 2.26x, 31B qat 2.38x, 12B fp8 1.69x.
- 정확도는 AR과 동일(전 셀 100%) — MTP는 무손실 기법(타깃 모델이 모든 토큰을 검증하므로 출력 분포 불변).
- **MoE는 이득이 작다**: 26B-A4B 1.6~1.7x, Qwen35-A3B 1.46x.

**왜 dense > MoE인가 (대역폭 논리):** dense는 매 토큰마다 전 파라미터(31B/12B)를 메모리에서 로드 →
디코드가 메모리 대역폭에 묶임. spec decode는 이 비싼 forward pass를 평균 수락길이(A)만큼 분할상환하므로
이득이 크다. MoE(활성 ~3~4B)는 이미 토큰당 로드량이 작아 대역폭 부담이 낮고, 분할상환 여지가 작다.

## 2. 양자화는 (이 태스크 한도 내) 정확도를 떨어뜨리지 않는다

- fp8 / int8 / qat-w4a16 전부 base(BF16)와 동일(100%).
- **단, 천장 효과 주의**: 이 태스크는 전 셀 100%로 포화 → "현저한 손상 없음"의 하한 보증일 뿐,
  미세 정확도 보존의 증거는 아니다(`01-results.md` 천장 효과 주의 참조).
- 속도만 단조 상승: 31B base **bf16 40.5 → fp8 67.4 → qat 89.6** tok/s.
- 실무 함의: 이런 명시적-신호 ops 태스크에선 정밀도를 낮춰도 능력 저하가 관측되지 않음 → 처리량·feasibility를
  안전하게 확보. 미세 정확도가 중요한 용도라면 더 어려운(천장에 안 닿는) 벤치로 재확인 필요.

## 3. diffusion — 속도 최강, 정확도는 유일하게 100% 미달

- 속도: 26B-fp8 diffusion **864 tok/s** (전 측정 중 최고). bf16 615 / int8 618.
- 정확도: 86.7/90.0/93.3% (= **26/27/28 of 30**) — 전부 AR(100%) 미달. **단 변형 간 차이는 N=30 노이즈라
  순위 무의미**(정밀도당 1문제 차, 95% CI ±11%p; `01-results.md` 각주4). "양자화가 정확도를 올린다"는
  물리적으로 기대 안 되는 방향이며 소표본 아티팩트.
- 트레이드오프: 최고 속도는 diffusion-fp8(864). 정확도는 diffusion 3종이 사실상 동급(86~93%, AR보다 낮음).
- diffusion int8 **지원 확인**(로드·측정 정상).

## 4. 속도-정확도 동시 최적 = Gemma 26B-A4B-fp8

| 기법 | 속도(short) | 정확도 |
|---|---|---|
| AR | 226 | 100% |
| AR+MTP | 390 | 100% |
| diffusion | 864 | 90% |

MoE의 적은 활성 파라미터(~4B)가 **처리량·메모리 feasibility 양쪽에 유리**. 정확도 100%가 필수면
AR 또는 AR+MTP(390 tok/s, 무손실), 최고 속도가 필요하고 90% 정확도 허용 가능하면 diffusion.

## 5. 한 장 요약

- **무손실로 빠르게:** MTP (dense 2.6x, 무손실). dense일수록 효과 큼.
- **메모리 아끼며 빠르게:** 양자화 (정확도 무손실, qat가 최속 base).
- **극단 처리량:** diffusion (864 tok/s, 단 정확도 90% 안팎).
- **단일 H100 한계:** 31B-bf16은 가중치만 62GB라 MTP 동시 적재 빠듯(`04-troubleshooting.md` #8).
