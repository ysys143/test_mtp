**lossless 검증 (확정)**: 두 갈래로 검증 — (a) greedy GPQA-30 A/B, (b) greedy 토큰동일성 + 대조군(같은 AR 재현).

(a) **정확도 A/B (n=30, underpowered)**: 31B-fp8 0.800→0.733(−2/30), Qwen27-fp8 0.800→0.833(+1/30). ±랜덤 → "체계적 손실 징후 없음"까지만.

(a2) **정확도 정식 — MTP-on GPQA-Diamond full 198, AR과 동일조건(thinking-on·max_gen_toks 32768·temp0.6/top_p0.95), 전 9셀 직접 실측** (n=30 A/B 대체):

| 셀 | lm MTP / AR (Δ) | insp MTP / AR (Δ) |
|---|---|---|
| 12B-bf16 | 0.657 / 0.662 (−0.005) | 0.611 / 0.652 (−0.041) |
| 12B-fp8 | 0.641 / 0.652 (−0.010) | 0.636 / 0.616 (+0.020) |
| 26B-bf16 | 0.773 / 0.763 (+0.010) | 0.717 / 0.727 (−0.010) |
| 26B-fp8 | 0.783 / 0.773 (+0.010) | 0.737 / 0.707 (+0.030) |
| 26B-int8 | 0.808 / 0.758 (+0.051) | 0.712 / 0.732 (−0.020) |
| 31B-fp8 | 0.818 / 0.849 (−0.030) | 0.747 / 0.763 (−0.016) |
| 31B-qat | 0.823 / 0.833 (−0.010) | 0.758 / 0.773 (−0.015) |
| qwen35-fp8 | 0.788 / 0.803 (−0.015) | 0.828 / 0.838 (−0.010) |
| qwen27-fp8 | 0.818 / 0.828 (−0.010) | 0.854 / 0.854 (0.000) |

→ **평균 Δ ≈ 0** (lm −0.001, insp −0.007; 셀당 표본오차 SE~0.03). 최대편차도 26B-int8 lm +0.051·12B-bf16 insp −0.041로 **부호 랜덤**(체계적 방향 없음). **n=198에서 MTP 정확도 = AR 정확도 확정** — n=30 A/B의 한계 해소.

(b) **토큰동일성 (greedy temp=0, 12프롬프트)** — MTP off-vs-on vs 대조군 off-vs-off2:

| | 대조군 off-vs-off2 | MTP off-vs-on |
|---|---|---|
| Gemma 31B-fp8(별도 드래프터) | **12/12 완전일치**(AR 결정적) | 3/12, 분기 늦음(tok 483~801) |
| Qwen27-fp8(임베디드 head) | 4/12, 분기 늦음(250~955)(AR 자체 비결정) | 0/12, 분기 이름(5~243) |

**판정**:
- **"토큰단위 무손실"은 반증** — greedy라도 MTP는 AR과 토큰열이 다름.
- **원인 = 검증 forward의 FP 비결정성**(편향 아님): Gemma는 AR이 12/12 결정적인데 MTP만 갈라짐(배치 검증의 FP 리덕션 순서차로 근소동률 argmax 뒤집힘), 분기가 **수백 토큰 뒤(late)** → 앞 장구간 일치 = 체계적 손실 징후 없음. Qwen27은 AR 자체가 이미 비결정(4/12)이고 MTP가 더 일찍 갈라짐(임베디드 head 섭동이 baseline보다 큼).
- **종합**: **분포-무손실(이론) 유지 + 체계적 정확도 손실 징후 없음(A/B ±랜덤·Gemma late-divergence)**. 단 **토큰단위로는 비결정적**이라 **"비트-무손실 확정"은 불가**. (사용자 우려 반영한 정확한 표현)

## D. 컨텍스트 길이 (NIAH, needle 검색 depth 10/50/90%)

| 모델군 | 측정 길이 | NIAH | 비고 |
|---|---|---|---|
| Gemma 12B / 26B-A4B / 31B | ~241K (cap 262144) | **3/3** | 전 depth 완벽 |
| Qwen35-A3B / Qwen27 | ~241K | **3/3** | 전 depth 완벽 |
| diffusion-26B-A4B | 32K / 48K | **2/3** | depth 90%(말단) 일관 실패(근처 로그 숫자 오답) |

- **AR(Gemma/Qwen)**: ~241K(cap 256K 근처)까지 전 depth 완벽 — 실효 컨텍스트 = 광고 256K에 부합.
- **diffusion**: 기존 16K 서빙을 넘어 **32K·48K까지 서빙·검색 가능**(canvas 고정 한계 아님). 단 **말단(90% depth) needle을 일관 놓침**(10%·50%는 성공) → 블록 디퓨전의 말단 회수 약점. (16384 셀은 출력 512토큰 여유분이 cap 초과 → HTTP400 하네스 아티팩트, 32K/48K로 대체.)
- **결론**: diffusion은 중간 길이 문서까지 다루나 **긴 문맥 말단 회수 신뢰도가 AR보다 낮음** — 하드추론 약점과 같은 결.

## E. 속도 측정 조건 (정본)

- **지표**: `bench_tp.py` — total throughput = completion_tokens / 전체요청시간(ms→s), `ignore_eos=True`·max_tokens=512·temp=0·warmup 1회·prefix-cache off. short(짧은 프롬프트)·8K(≈8K 입력) 2조건.
- **서빙 조건(셀별 최적·실배포 기준)**: AR 셀 max-model-len 40960 / util 0.92 (31B-bf16은 +kv-fp8). MTP 셀 max-model-len 16384 / util 0.85(드래프터 여유). qwen35-bf16·int8은 70GB로 cudagraph 불가 → **enforce-eager 폴백(15 tok/s — 공정 측정치 아님, 단일 H100 메모리 제약 산물; 표 A †주)**. diffusion :gemma 이미지 / max-num-seqs 4. 전 셀 동일 H100 80GB 단일 spot.
- 속도표(A/B/C)는 모두 위 동일 하네스 산출 → 상호 비교가능. 서빙조건은 "각 셀이 실제로 가장 잘 도는 설정"으로 통일하지 않고 명시(균일화는 일부 셀을 비최적으로 만들어 오히려 비현실적).

## 핵심 발견 (4대 질문 답)

`★ Q2 양자화가 정확도를 떨어뜨리나? → 아니오 ────`
- bf16/fp8/int8/qat 정확도 차 ±0.02~0.05(노이즈). 속도만 향상: 12B qat 1.6×, 31B qat 90 vs bf16 40 등. dense·MoE·전 크기 일관. **단 Qwen27은 int8 속도이득 0**(선형어텐션 recurrence 병목, 가중치 대역폭 아님).
- **qwen35-A3B 정밀도 무손실 최종확인**: GPQA bf16 0.808 / fp8 0.803 / int8 0.788(±0.02). **단 이 하드웨어(단일 H100)에선 실용 정밀도가 fp8 뿐** — bf16/int8(70GB)은 메모리가 빠듯해 cudagraph 못 올리고 enforce-eager로 **폴백** → 15 tok/s(fp8 217의 1/14). 이 15는 정밀도의 본질 성능이 아니라 **단일 H100 폴백값**이며, 메모리 여유 환경에선 graph 적재로 정상화된다(표 A †주 참조). 함의: 정확도는 무손실이라도 **graph 적재 가능 여부가 비용을 가른다**.
`────────────────────────────────────────`

`★ Q1·Q4 MTP 이득? MoE vs dense? ─────────────`
- **dense가 이득 최대**(31B-fp8 2.62×, 12B-bf16 2.24×): AR이 느려 spec-decode 여지 큼.
- **MoE는 작음**(Qwen35 1.43×, 26B 1.6~1.75×): 이미 빨라 headroom 적음.
- **정확도**: **MTP-on GPQA full 198을 전 9셀 직접 실측 → 평균 Δ≈0(lm −0.001/insp −0.007), AR과 동일 확정**(C-a2). 토큰동일성은 "비트-무손실"만 반증(greedy도 MTP≠AR, 원인=검증 forward FP 비결정성·편향 아님). 즉 **정확도 무손실은 정식 확정, 토큰열은 비결정**. 31B-bf16만 드래프터+KV OOM으로 불가.
`────────────────────────────────────────`

`★ Q3 diffusion 속도/정확도? → 빠르지만 과제별 ──`
- **최速(864 tok/s, AR 26B-A4B의 ~4×)**, 양자화도 무손실.
- **정확도 손실이 추론깊이에 집중**: GPQA −0.20, MMLU-Pro −0.09, **한국어 haerae −0.05·IFEval −0.03(거의 보존)**. → 언어이해·지시준수엔 거의 무손해로 4배 빠름, 하드추론엔 부적합.
- **컨텍스트(NIAH)**: 32K·48K까지 서빙·검색되나 **말단(90% depth) 회수 실패(2/3)** — AR의 240K 3/3 대비 긴문맥 신뢰도 낮음(D절).
`────────────────────────────────────────`

`★ 속도×정확도 프론티어 ──────────────────────`
- **활성 파라미터 적은 MoE 우위**: Qwen35-A3B(~3B, 217)·26B-A4B(~4B, 226)가 dense 31B(40~90)보다 3~5× 빠르고 MMLU-Pro 동급(0.86). dense 31B=정확도 천장(GPQA 0.85·IFEval 0.944)·최저속.
