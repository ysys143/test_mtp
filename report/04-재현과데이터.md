## 3. 재현 패턴

```bash
# 1) 서빙 (AR): nightly, prefix-cache off, reasoning-parser
docker run -d --name gd --gpus all --ipc=host \
  -v ~/.cache/huggingface:/root/.cache/huggingface \
  -e VLLM_MEMORY_PROFILER_ESTIMATE_CUDAGRAPHS=0 -p 8000:8000 \
  vllm/vllm-openai:nightly --model google/gemma-4-31B-it \
  --max-model-len 40960 --gpu-memory-utilization 0.92 \
  --no-enable-prefix-caching --reasoning-parser gemma4 [--quantization fp8] [--kv-cache-dtype fp8]

# 2) thinking 프록시
ENABLE_THINK=true setsid python3 inject_proxy.py &   # :8001 → :8000

# 3) MTP: --speculative-config 추가 (Gemma=별도 드래프터, Qwen=임베디드 head)
#   '{"model":"google/gemma-4-31B-it-assistant","num_speculative_tokens":4}'
#   '{"method":"mtp","num_speculative_tokens":2}'

# 4) 평가 (전부 base_url=:8001 경유, max_gen_toks=32768, temp0.6/top_p0.95)
#   lm-eval GPQA(full) / inspect mmlu_pro·ifeval(--epochs 1) / run_hret.py haerae·kmmlu

# 5) 속도 / 컨텍스트
python3 bench_tp.py <label>          # short + 8K
python3 niah.py <label> 0 0.1,0.5,0.9
```

**재개 가능 설계:** 셀·벤치별 `$CAMP/.done_*` 마커로 skip, 셀별 즉시 CSV append, 캠페인은 `setsid`로 detached·파일로깅(Spot 선점/sshd wedge 대비). 폴링은 가벼운 sudo 읽기만.

**메모리 가이드(H100 80GB):** dense 31B bf16는 kv-fp8 필수(아니면 94K cap), MTP는 OOM 경계. fp8/qat는 util 0.90 여유. diffusion `--max-num-seqs 4`. qwen35 bf16/int8은 graph 불가 → eager(15tok/s, 비교 무효).
