# 05 — 2부 리그 (신규 6모델 경량 평가)

> **무엇을·왜:** 2026년 신규 서드파티·distill·finetune 모델 6종을 1부(Gemma4/Qwen3.6) 옆 **경량 "2부" 티어**로 평가. 풀 배터리가 아니라 **GPQA + MMLU-Pro + KMMLU**(thinking-on 동등조건, 단일 정밀도, 속도)만. 6종의 서빙이 극히 이질적(MoE/VL/Omni-멀티모달/Mamba하이브리드/GGUF/distill)이라, **"서빙을 먼저 완벽히 안정화 후 측정"** 원칙으로 진행. 데이터: `../results_consolidated.csv`(2부 셀).

## B. 해석

1. **EXAONE-4.5 / Nex-N2 = "지식·한국어 1부급, 하드추론은 중간"**
   - EXAONE MMLU-Pro **0.858**·KMMLU **0.675**는 2부 최고이자 **1부 top과 동급**(LG 모델답게 한국어 최강). 단 GPQA 0.556으로 하드 과학추론은 한 수 아래.
   - Nex-N2도 동일 결: MMLU-Pro 0.836·KMMLU 0.634(1부급), GPQA 0.747(1부 26B와 12B 사이). 6종 중 GPQA 1위.
   - → 두 모델은 **광범위 지식은 강한데 GPQA(초고난도 추론)만 프론티어에 못 미치는** 공통 프로파일.

2. **VibeThinker-3B = 특화의 극단** — 3B가 **GPQA 0.535로 자기보다 10배 큰 모델들과 동급**(STEM 추론 RL 특화). 그러나 MMLU-Pro 0.310·KMMLU 0.266로 광범위지식·한국어는 전무. "좁고 깊은 칼": 특정 영역(수학/STEM)만 대형급, 일반지식은 3B 수준.

3. **Claude-distill = 화제성 distill이 오히려 손해** — Qwen3.6-35B-A3B를 Claude Opus 추론으로 distill한 모델이 **base(1부 qwen35 0.80/0.86/0.59) 대비 전 벤치 하락**(0.515/0.756/0.518). attention-only LoRA distill이 base의 강한 추론을 평탄화. "유명 base + 화제 distill ≠ 개선"의 실측 사례.

4. **나머지(GLM-Flash·Nemotron) = 경량·고속 등급 trade-off** — GPQA 0.45~0.55로 1부 12B(0.65)에도 미달. 속도(GLM 177, Nemotron 331)와 맞바꾼 포지션.

5. **한국어** — EXAONE(0.675)·Nex-N2(0.634) 빼면 전멸(0.43~0.52), 1부 Gemma(0.64~0.71) 미달.

**한 줄 결론:** 이번 2부 6종 중 실전 후보는 **EXAONE-4.5·Nex-N2 둘**(지식·한국어 강), VibeThinker는 **STEM 추론 전용**, distill·Nemotron·GLM-Flash는 용도 한정/반면교사.

## C. 서빙 안정화 R&D (이 작업의 핵심 — "먼저 안정화" 원칙)

6종 모두 서빙 방식이 달라 모델별로 연구·안정화했다. nightly(0.22.1dev)엔 신규 파서·아키가 없어 **vLLM 0.23.0(latest)로 갱신**한 게 다수의 열쇠.

| 모델 | 런타임 | parser | 정밀도 | 안정화 열쇠 |
|---|---|---|---|---|
| qwendistill | vLLM nightly | qwen3 | online fp8 | 1부 qwen35 경로 재사용 |
| GLM-4.7-Flash | **vLLM 0.23.0** | deepseek_r1 | online fp8 | 새 이미지(nightly엔 glm 파서·아키 없음); `<think>` 미출력 → inline 추론 |
| EXAONE-4.5 | vLLM 0.23.0 | qwen3 | online fp8 | **`--limit-mm-per-prompt 0`**(VL 비전블록 버그 `sequence_lengths` 우회, 텍스트 서빙) + trust-remote-code |
| Nemotron-Omni | vLLM 0.23.0 | nemotron_v3 | **native fp8**(33GB) | trust-remote-code + mm 비활성 + `--kv-cache-dtype fp8` |
| Nex-N2-mini | vLLM 0.23.0 | qwen3 | online fp8 | mm 비활성 (카드의 sglang 권장 불필요 — vLLM로 로드됨) |
| gemma-4-12B-coder | **불가** | — | GGUF Q8_0 | vLLM GGUF가 `gemma4_unified` 레이어별 KV(`num_key_value_heads`=list) 거부 → llama.cpp 필요, **제외** |

**공통:** thinking-on은 inject_proxy(:8001) 주입 — 단 distill/GLM/EXAONE/Nemotron/Nex-N2 모두 `<think>` 태그 없이 **content에 inline 추론** → reasoning-parser는 no-op이나 flexible-extract/마지막추출로 채점 무결(스모크로 검증).

### 방법론 교훈 (실패에서)
- **EXAONE 과대생성 → inspect 재시도 폭주**: EXAONE는 MMLU-Pro에서 문항당 ~10k+ 토큰(일부 32768 cap)을 생성 → 개별 요청이 inspect 타임아웃 초과 → 재시도 폭주(N=500에서 5.5h 무수렴, N=200에서 RetryError abort). **해결 = inspect `--timeout 3600 --max-retries 3 --fail-on-error 0.3`**(개별 장문 허용 + 일부 실패해도 abort 금지). 이후 N=200으로 0.858 정상 수렴(3/200 미채점).
- **하네스 일관성**: EXAONE MMLU-Pro를 lm-eval로 우회하려다 — 타 5모델이 inspect라 **하네스가 다르면 비교 불가** → inspect로 통일(위 플래그로 완주). 단 EXAONE만 N=200(각주).
- **진행 가시성 한계**: inspect는 완료 시 일괄 기록이라 실시간 진행률·부분점수가 안 보임. 향후 진행 가시성이 중요하면 lm-eval(진행바)와의 트레이드오프 고려.
- **디스크 회전**: 6모델 순차(모델당 ≤72GB) — 1부 캐시 회전 + 모델별 측정 후 즉시 삭제로 단일 H100 디스크(485GB) 내 운용.

## D. 1부 대비 종합
- **2부 최강 2종(EXAONE·Nex-N2)도 GPQA에선 1부 강자(0.80~0.85)에 못 미침** — 하드추론은 여전히 Gemma4/Qwen3.6(1부)이 우위.
- **지식(MMLU-Pro)·한국어는 EXAONE·Nex-N2가 1부와 동급** 도달 — 이 축에선 2부 신모델이 따라잡음.
- 속도는 활성 파라미터 적은 MoE/하이브리드(Nemotron 331, Nex-N2/qwendistill 211)가 dense(EXAONE 70)를 압도 — 1부 결론(저활성 MoE 우위)과 일관.
